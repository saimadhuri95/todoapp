# Release Checklists

Each release gets its own manual verification checklist in this folder.

## How to use this folder

1. Copy the most recent checklist file.
2. Rename it for the release you are preparing.
3. Work through every item on real devices / real installs.
4. Keep failures, notes, and follow-up fixes in the release file itself.

## Current checklist

- `v0.1.0-checklist.md` - baseline matrix for alarms, pairing, permissions,
  cloud-folder sync, and reboot behavior.
