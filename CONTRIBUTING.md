## Pull Requests

- Keep changes focused and easy to review.
- Run `dart format --output=none --set-exit-if-changed lib test` before
  opening a PR when Dart files change.
- Run `flutter analyze` and `flutter test` when the Flutter toolchain is
  available.
- Update docs or tests when behavior changes.
